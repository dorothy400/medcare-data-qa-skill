# 制图规范（chart-conventions）

本规范适用于 medcare-data-qa skill 在飞书机器人回答中生成的所有图表（柱状图、折线图、双轴图等）。**当回答里包含任何图表时，必须遵循以下规范。**

所有图表必须通过飞书原生 `chart` 元素渲染（基于 VChart），不允许用 plotly / matplotlib / seaborn 生成 PNG 再上传——详见 SKILL.md "🚫 严禁 plotly..." 节。

---

## 1. 颜色规则

**核心默认：不显式指定 `color` / `color_theme`，让飞书 VChart 用默认 brand 主题自动配色。** 飞书 VChart 的默认色板已经是专门为飞书卡片设计的，色彩搭配 / 区分度 / 对比度都经过设计验证，比 worker 即兴选色稳定得多。

### 1.1 一般情形（默认路径）：交给飞书 VChart 默认色板

适用所有情形：单系列 / 多系列 / 离散分类 x 轴 / 时间 x 轴 / 多分组叠加。

```json
{
  "tag": "chart",
  "chart_spec": {
    "type": "bar",
    "data": { "values": [...] },
    "xField": "month",
    "yField": "amount",
    "seriesField": "region"
    /* ↑ 不设 color，不设 color_theme，由 VChart 默认 brand 主题处理 */
  }
}
```

VChart 会根据 `seriesField` 的不同取值，自动从默认色板分配连续槽位。多个区域 / 部门 / 产品线在同一张图上时，每个会自动得到一个不同的颜色，不需要 worker 干预。

**禁止做法**：
- 显式指定 `color: ["#xxx", ...]` 数组（**除非走下面的 § 1.2 双轴例外**）
- 显式指定 `color_theme: "rainbow"` / `"complementary"` 等非默认主题
- 用同色族深浅区分**不同分组**（如蓝 + 浅蓝区分两个区域）—— VChart 默认色板已经处理好这件事，worker 不要画蛇添足

### 1.2 双轴图例外：同分组的不同指标用深浅区分

**唯一需要 worker 显式控制颜色的场景**：双轴图同时展示一个分组的**两个指标类型**（如"亚洲的差值柱 + 亚洲的环比%折线"）。

规则：
- **不同分组**之间：仍由 VChart 默认色板分配（不要手动指定）
- **同一分组的两个指标**：用**同一色相的深浅两个色**区分。柱用浅色（基础色），折线用同色相的**深色变体**（手动指定 hex，明度降 25-35%）

例：双轴图展示 "亚洲（除日韩） + 日韩" 的差值柱 + 环比%折线，整张图有 4 个系列：
- 亚洲（除日韩）差值柱 → VChart 默认色板第 1 色（假设是某种蓝）
- 亚洲（除日韩）环比%折线 → 上面那种蓝的**深色变体**
- 日韩差值柱 → VChart 默认色板第 2 色（假设是某种橙）
- 日韩环比%折线 → 上面那种橙的**深色变体**

**实施方式**：worker 在 chart_spec 里给每个 series 单独配 `style.fill`（柱）或 `line.stroke`（折线）。在 4 系列双轴图里，柱可以省略 fill 让 VChart 默认配色；折线必须手动指定 `stroke` 为同色族深色版本。

这条规则保证：
- 用户看图例 4 项时，能按"颜色相近 = 同一分组"分组扫读
- 同分组内柱 / 折线又能立即区分"哪个是差值哪个是百分比"

## 2. 多个分组 → 必须画在同一张图，不允许拆图

当数据涉及多个分组维度（多个区域、产品线、组织、渠道等）时：

- **必须把所有分组画在同一张图上**，用 § 1.1 VChart 默认色板按 `seriesField` 自动分色。
- 不允许"亚洲（除日韩）一张图、日韩一张图"这种按分组拆分的做法，即使它们各自看起来更清爽。
- 拆图会切断用户在多个分组间做"快速对比"的能力——这是用户问"X 和 Y"时的核心诉求，必须保留。
- 适用于柱状图、折线图、面积图、**双轴图**。例如双轴图展示"亚洲（除日韩） + 日韩"的差值+环比%，整张图上有 4 个系列：
  - 亚洲（除日韩）差值（VChart 默认第 1 色，柱）
  - 亚洲（除日韩）环比 %（第 1 色的深色变体，折线）
  - 日韩差值（VChart 默认第 2 色，柱）
  - 日韩环比 %（第 2 色的深色变体，折线）

**例外**：仅当分组数 > 6 且视觉确实拥挤时（如 8 个区域的月度折线在小卡片图里相互遮挡），允许拆成 2 张图（4+4 或 3+5），但每张图内仍把多个分组叠加，**不允许**拆到每个分组一张图。

### 拆图策略（复杂查询：按时间粒度 / 按指标类型拆，绝不按分组拆）

当问题涉及**多个时间粒度**（月度 + 年度）、**多个指标类型**（绝对值 + 环比% + 同比%）、或两者叠加时，一张 VChart 图塞不下、视觉会乱。**正确拆法**是拆成多张 VChart 子图，**全部放在同一张 interactive card 的 `body.elements` 里**，子图之间用 markdown 小标题分隔。

允许的拆图轴：
- **按时间粒度拆**：一张"月度趋势"折线 + 一张"年度合计"柱状
- **按指标类型拆**：一张"绝对值"柱 + 一张"环比%"折线 + 一张"同比%"折线

绝对禁止的拆图轴：
- ❌ **按分组维度拆**——破坏 § 2 的"多分组同图对比"原则

例：用户问"客户 A & B 近三年同期接单分析"，正确输出 3 张 VChart 子图：
1. **月度订单趋势**（折线图）：2 个客户作为 series，按 24/25/26 三年的月份展示
2. **年度合计对比**（折线图）：2 个客户作为 series，横轴 24/25/26 三年
3. **环比 / 同比%**（折线图）：2 个客户的 mom% / yoy% 趋势

⛔ **严禁用 plotly / matplotlib 兜底**——再复杂的查询也不允许跑回 PNG 路径。如果实在画不动，宁可少画一两张子图（在 markdown 里用表格代替），也不准 PNG。详见 SKILL.md "🚫 严禁 plotly..." 节。

## 3. 目标 + 实际 + 完成率 → 必须用并列柱 + 折线双轴图

当回答**同时**展示目标金额、实际达成金额和完成率（达成率）时（最常见于"完成率 / 达成率 / 完成情况"类查询走 JOIN 后的输出）：

- **必须**合并到一张图，**不允许**拆成"目标一张图、实际一张图、完成率一张图"
- 必须是**双轴图**：
  - **左轴**：金额（USD），同时承载目标柱 + 实际柱
  - **右轴**：完成率（%），用折线 + 数据点 marker
- **目标 vs 实际必须并列（grouped bars / side-by-side）**，**严禁堆叠（stacked）**——堆叠会让用户难以直接对比两者高度差，而"差距"恰恰是用户想看的核心信息
- **完成率折线必须用百分比格式**（`87%`、`120%`），而不是 `0.87` 或 `1.2`，方便用户一眼判断"是否大于 100%"
- **零点对齐**：左右轴的 0 必须在同一水平线上（参见 § 4）

例：某部门 5 月接单目标 vs 实际：
- 一部：目标柱 + 实际柱并列；完成率折线点 47.6%
- 二部：同上；完成率折线点 40.7%
- 三部：同上；完成率折线点 30.4%

横轴 = 部门（离散分类），左轴 USD（柱），右轴 %（折线）。所有部门连成一条完成率折线，方便横向对比哪个部门最接近达成。

### 🎯 标准 VChart spec 模板（直接照抄，三处替换数据即可）

**注意**：worker 之前犯过的错——用 `type: "bar"` + 单条 series + 没有 axes 配置，VChart 默认会**堆叠**，且**折线没法挂右轴**。必须用下面这个 `type: "common"` 的多 series + 双 axes 写法：

```json
{
  "tag": "chart",
  "aspect_ratio": "16:9",
  "preview": true,
  "chart_spec": {
    "type": "common",
    "series": [
      {
        "type": "bar",
        "id": "bar",
        "xField": ["dept", "metric"],
        "yField": "amount",
        "seriesField": "metric",
        "data": {
          "values": [
            { "dept": "一部", "metric": "目标", "amount": 4528275, "amount_label": "$4.5M" },
            { "dept": "一部", "metric": "实际", "amount": 2156573, "amount_label": "$2.2M" },
            { "dept": "二部", "metric": "目标", "amount": 1980611, "amount_label": "$2.0M" },
            { "dept": "二部", "metric": "实际", "amount": 806645,  "amount_label": "$807K" },
            { "dept": "三部", "metric": "目标", "amount": 1648346, "amount_label": "$1.6M" },
            { "dept": "三部", "metric": "实际", "amount": 501394,  "amount_label": "$501K" }
          ]
        },
        "label": { "visible": true, "formatter": "{amount_label}" }
      },
      {
        "type": "line",
        "id": "line",
        "name": "达成率",
        "xField": "dept",
        "yField": "rate",
        "data": {
          "values": [
            { "dept": "一部", "rate": 47.6 },
            { "dept": "二部", "rate": 40.7 },
            { "dept": "三部", "rate": 30.4 }
          ]
        },
        "point": { "visible": true, "style": { "size": 8 } },
        "label": { "visible": true, "formatter": "{rate}%" }
      }
    ],
    "axes": [
      { "orient": "left",   "type": "linear", "seriesIndex": [0], "title": { "visible": true, "text": "金额 (USD)" } },
      { "orient": "right",  "type": "linear", "seriesId": ["line"], "grid": { "visible": false }, "title": { "visible": true, "text": "达成率 (%)" }, "min": 0, "max": 150 },
      { "orient": "bottom", "type": "band",   "label": { "visible": true } }
    ],
    "legends": { "visible": true, "orient": "bottom" }
  }
}
```

**关键不能漏的 7 处**：

| # | 字段 | 作用 |
|---|---|---|
| 1 | `"type": "common"` | 顶层 chart 类型必须是 common（混合图），不能是 bar |
| 2 | `series[0].xField: ["dept", "metric"]` | **数组形式**让柱状图变成并列分组（grouped），不是堆叠 |
| 3 | `series[0].seriesField: "metric"` | 告诉 VChart 把 metric 字段（"目标"/"实际"）作为分组依据 |
| 4 | `series[1].name: "达成率"` | **必须显式指定**折线名，否则图例显示 "Line" 而不是 "达成率" |
| 5 | `axes[1]` 右轴 + `seriesId: ["line"]` | 把折线（id=line）绑定到右轴。**不写这一步折线就掉到底部用左轴** |
| 6 | `axes[1].min: 0, max: 150` | 给完成率轴限定 0-150% 范围 |
| 7 | 数据值预格式化为 `amount_label` 字段 | 金额单位 K/M 转换在 SQL/JS 侧预算好，柱标签用 `formatter: "{amount_label}"` 渲染。规则见下方"金额格式化"节 |

**金额格式化（K / M / B 自动缩写）—— 仅用于图表，不用于表格**：

**作用范围**：本规则**只适用于 `chart` 元素**里的数据标签、轴刻度。在 `table` 元素里**严禁**用 K/M/B 缩写——表格的核心价值是"用户可以复制完整数字到 Excel 做分析"，缩写会丢精度，破坏这个用途。

**图表（chart）里的金额标签必须缩写**——直接显示 `4528275` 看起来很乱、轴刻度也压不下。规则：

| 原始值范围 | 缩写规则 | 例 |
|---|---|---|
| `< 1,000` | 保留原值 | `$842` |
| `1,000 ~ 999,999` | 除 1,000 后加 `K`，保留 0 位小数 | `$842K` |
| `1,000,000 ~ 999,999,999` | 除 1,000,000 后加 `M`，保留 1 位小数 | `$4.5M` |
| `≥ 1,000,000,000` | 除 1,000,000,000 后加 `B`，保留 2 位小数 | `$1.23B` |

实施方式：worker 在构造 chart_spec 时，**在 SQL 输出/Python 端预计算好** `amount_label` 字段，写入数据里。然后 chart 系列的 label 用 `formatter: "{amount_label}"` 渲染。

```python
# Python 端预格式化示例
def fmt_amount(n):
    if n is None: return ""
    n = float(n)
    if abs(n) >= 1e9: return f"${n/1e9:.2f}B"
    if abs(n) >= 1e6: return f"${n/1e6:.1f}M"
    if abs(n) >= 1e3: return f"${n/1e3:.0f}K"
    return f"${n:.0f}"
```

**表格（table）里的金额保持完整数字 + 千分位**，由 table 元素自身的 `data_type: "number"` + `format` 渲染：
```json
{ "name": "amount", "display_name": "目标(USD)", "data_type": "number",
  "format": { "symbol": "$", "precision": 0, "separator": true } }
```
渲染结果：`$4,528,275`（保留完整精度、千分位逗号）。

百分比直接显示 `{rate}%` 即可，图表和表格都不缩写。

## 4. 差值 + 差值百分比 → 必须用双轴图

当回答中**同时**展示差值（绝对差）和差值百分比时：

- **必须**合并到一张双轴图，而不是分成两张图。
- **左轴**：差值（绝对值，单位与本期值一致，如 USD）。
- **右轴**：差值百分比（%）。
- **零点对齐**：左右轴的 0 必须在同一水平线上。具体做法是让左右轴的 min/max 对称——如果差值范围是 `[-1000, 3000]`，则左轴设为 `[-3000, 3000]`；右轴的 min/max 也按同样对称比例计算（如 `[-20%, 20%]`）。这样正负分界线视觉上重合。
- 不允许出现"左轴零点在中间、右轴零点在下三分之一"这种错位。

---

## 5. 双轴图中差值用柱、百分比用折线

- **差值**：柱状图（bar / column）。柱状图直观传达"加减了多少"。
- **差值百分比**：折线图（line + markers）。折线图直观传达"变化趋势/斜率"。
- 不允许把两者都画成柱，也不允许差值用折线、百分比用柱。

---

## 6. 折线颜色必须比柱颜色深（同分组内）

双轴图中柱和折线会在视觉上**叠加**。如果两者颜色明度相近，折线会被柱"吃掉"或边缘融化。规则：

- **折线使用同分组柱色的"深色变体"**（明度降 25-35%）。
- 折线宽度建议 `2.5–3 px`，圆点 marker 直径 `5–6 px`，让它在柱面前依然清晰。
- 不同分组之间柱和折线的色族关系参考 § 1.2。

---

## 7. 时间维度必须放在横轴

当用户问题中出现以下时间粒度词时，**必须**把对应时间维度作为图表的横轴（x-axis）：

| 用户用词 | 横轴粒度 | 横轴标签格式 |
|---|---|---|
| 每个月 / 每月 / 各月 / 按月 / monthly | 月 | `2026-01` / `2026-02` …（或 `1月` / `2月`） |
| 每季度 / 各季度 / 按季度 / quarterly | 季度 | `2026 Q1` / `2026 Q2` … |
| 每周 / 各周 / 按周 / weekly | 周 | `2026-W18`（ISO 周）或 `5/4–5/10` |
| 每天 / 每日 / 按日 / daily | 日 | `2026-05-15` 或 `5/15` |
| 每年 / 各年 / 按年 / yearly | 年 | `2024` / `2025` / `2026` |

规则：
- **横轴=时间，纵轴=指标值**，不要反过来（绝对禁止把月份放纵轴、销售目标放横轴）。
- 时间点按**自然时间顺序**从左到右排列，不允许按数值大小排序、不允许打乱。
- 分组维度的处理见 § 2（多分组必须叠加在同一张图上）。
- 时间点缺失（如 2 月没数据）→ 折线图断开（不连接），柱状图留空，**不要静默插值或对齐到 0**。
- 双轴图（§ 3-6）适用此规则：横轴依然是时间，左轴金额或差值（柱）、右轴完成率或百分比（折线）。

---

## 8. 其他通用约束

- **轴标签必须有单位**：左轴标"销售目标 (USD)"或"差值 (USD)"，右轴标"环比 (%)"。不要只写"金额"。
- **百分比格式**：右轴刻度统一用 `XX%` 形式（如 `12%` 而不是 `0.12` 或 `12`）。
- **数据标签**：双轴图的折线点上标百分比数值（`+12.4%` / `−3.1%`），柱顶标差值绝对值（`+1.2M` / `−340K`）。两者都要带正负号。
- **零基准线**：双轴图必须画一条 y=0 的水平虚线（灰色 `#999999`，宽度 1 px），强化零点对齐的视觉效果。
- **配色一致性高于美观**：若用户提了自定义颜色诉求，遵循用户；否则走 § 1 规则，不要"换个好看的"。

---

## 9. 何时**不**画图表

- 单个数值（如"上周销售目标 = 1.2M"）→ 直接卡片显示，不画图。
- 行数 ≤ 2 的结果 → 表格即可，不画图。
- 时间序列只有 1 个点 → 卡片或表格，不画图。

---

## 10. 实施提示（生成图表时）

- 大部分场景**不需要**在 chart_spec 里写 `color` 或 `color_theme`，VChart 默认色板已经处理好分色 / 对比度。
- 唯一需要手工 hex 的场景是 § 1.2 双轴图同分组深浅区分（手动指定折线 stroke 为柱色的深色变体）。
- 如果某次输出 worker 想偏离上述规则（比如想用 rainbow 主题），必须在卡片里向用户说明理由。
